1) What this assignment is about?

This assignment implements the Berkeley Clock Synchronization Algorithm in a distributed system.

The goal is:

synchronize clocks of multiple systems
make all client systems maintain nearly same time

In distributed systems, every computer has its own clock, and clocks may become different due to:

network delay
hardware differences
clock drift

Berkeley Algorithm synchronizes all clocks by calculating an average time.

The lab manual explains Berkeley Algorithm is a clock synchronization technique used in distributed systems where a master system calculates average time and synchronizes all client clocks.

What problem does this solve?

Suppose:

Server time = 10:00:05
Client1 time = 10:00:10
Client2 time = 09:59:58

Different times may cause:

incorrect transaction order
synchronization problems
logging issues

Berkeley Algorithm solves this by making all systems agree on one common time.

What is Berkeley Algorithm?
Definition

Berkeley Algorithm is a clock synchronization algorithm used in distributed systems where:

one machine acts as master/server
other machines act as clients/slaves
master collects time from all clients
average time is calculated
time adjustments are sent to all clients
Main Idea

Instead of using internet time:

All systems agree on average time
Real-life Analogy

Suppose:
5 friends have different watch times.

One friend:

asks everyone's time
calculates average
tells everyone how much to adjust

Now all watches show nearly same time.

That is Berkeley Algorithm.

Components in Berkeley Algorithm
Component	Role
Server/Master	Collects times
Clients	Send local times
Average Time	Common synchronized time
Adjustment	Difference sent to clients

Server acts as master.

Main Method

Program execution starts here.

Create Server Socket
ServerSocket serverSocket = new ServerSocket(5000);
Purpose

Starts server on port:

5000

Server waits for client connections.

Print Message
System.out.println("Server started...");

Displays server startup message.

Store Clients
List<Socket> clients = new ArrayList<>();

Stores connected client sockets.

Store Times
List<Long> times = new ArrayList<>();

Stores client clock times.

Time Formatter
SimpleDateFormat sdf = new SimpleDateFormat("HH:mm:ss");

Formats time in:

hours:minutes:seconds
Accept Clients
for(int i = 0; i < 3; i++)

Accepts:

3 clients
Accept Connection
Socket socket = serverSocket.accept();

Waits until client connects.

Store Socket
clients.add(socket);

Stores connected client.

Receive Client Time
DataInputStream in = new DataInputStream(s.getInputStream());

Receives data from client.

Read Time
long clientTime = in.readLong();

Reads client clock value.

Store Time
times.add(clientTime);

Stores received time.

Print Client Time
sdf.format(new Date(clientTime))

Converts milliseconds into readable time.

Server Time
long serverTime = System.currentTimeMillis();

Gets current server clock time.

Add Server Time
times.add(serverTime);

Server time also participates in averaging.

Calculate Sum
for(long t : times) sum += t;

Adds all times.

Calculate Average
long avg = sum / times.size();

Finds average clock time.

Send Adjustment
long diff = avg - times.get(i);

Calculates:

Adjustment = Average - Client Time
Send Adjustment to Client
out.writeLong(diff);

Sends time correction.

Close Server
serverSocket.close();

Stops server.

Q1) What is Berkeley Algorithm?

Berkeley Algorithm is a clock synchronization algorithm where a master system collects times from clients, calculates average time, and sends adjustments to synchronize clocks.

Clock synchronization is needed to maintain consistency and proper event ordering in distributed systems.

Clock drift means clocks of different systems gradually become different over time.

Average time helps all systems synchronize to a common time fairly.
