1) What this program is about?
This program implements multi-threaded client-server communication using Java RMI in Distributed Systems.
It allows multiple clients to communicate with a remote server simultaneously.
Simple Explanation


Client sends a message to server


Server receives the message


Server processes request


Server returns response back to client


In this program, many client threads send requests at the same time, and Java RMI automatically creates separate threads on the server side to handle each client independently.
This practical is part of Distributed Systems Lab Assignment 1. 

What is Java RMI?
RMI = Remote Method Invocation
RMI is a Java API used to call methods of an object that exists on another machine/JVM.
Normally:
obj.method();
works locally.
With RMI:


client calls method remotely


method executes on server machine


result comes back to client


It feels like calling a local method, but actually communication happens over the network.

What is rmiregistry and why is it used?
Simple Definition
rmiregistry is a naming service in Java RMI.
It stores:
Name  -> Remote Object Reference
Example:
"MyServer" -> Remote Server Object

Why it is needed?
Client does not know:


where server object exists


memory address of object


So client asks registry:
"Give me object named MyServer"
Registry returns remote object reference.
Then client can call remote methods.

Real-life analogy
Think of rmiregistry like:
Mobile Contacts List
You save:
Mom -> Phone Number
Similarly:
MyServer -> Remote Object
Client searches by name instead of remembering address.
The manual also defines RMI Registry as a bootstrap naming service used to bind remote objects and allow clients to look them up.

4) Viva Questions (Most Important)
Q1) What is RMI?
Answer:
RMI stands for Remote Method Invocation.
It allows a Java object on one machine to call methods of an object running on another machine.

Q2) Why do we use rmiregistry?
Answer:
rmiregistry is used to store and locate remote objects using names.
Client uses lookup() to get remote object reference.

Q3) Why does remote interface extend Remote?
Answer:
To indicate that methods can be invoked remotely through RMI.

Q4) Why is RemoteException used?
Answer:
Because network communication may fail during remote method calls.

Q5) How is multi-threading achieved here?
Answer:
Multiple client threads send requests simultaneously, and Java RMI automatically creates separate server threads for handling each request.
