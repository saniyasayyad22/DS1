1) What this program is about?

This program implements a distributed calculator application using CORBA.

The client sends arithmetic requests like:

Addition
Subtraction
Multiplication
Division

to a remote server.

The server processes the operation and returns the result to the client.

So even though the calculator methods exist on another machine/server, the client can use them as if they are local methods.

This practical belongs to CORBA-based distributed communication.

What is CORBA?
CORBA = Common Object Request Broker Architecture

CORBA is a distributed object technology used for communication between applications running on:

different machines
different operating systems
different programming languages
Simple Definition

CORBA allows:

Client object ↔ Remote server object

to communicate over network.

Why CORBA is used?

CORBA is used because:

distributed systems contain different technologies
systems may be written in Java, C++, Python etc.
CORBA provides communication between them
Main Goal of CORBA

"Make distributed objects communicate transparently."

Client does not need to know:

where server exists
which OS server uses
which language server uses

CORBA handles everything internally.

Real-life Example

Suppose:

Banking frontend written in Java
Transaction engine written in C++
Database service written in Python

CORBA allows all of them to communicate.

What is Object Brokering?
Broker Meaning

A broker acts like a mediator.

In CORBA:

ORB acts as broker

ORB = Object Request Broker

It transfers requests between:

Client ↔ Server Object
Real-life analogy

Think of ORB like:

Swiggy/Zomato Delivery Partner

You:

place order

Delivery partner:

carries request to restaurant
brings response back

Similarly ORB:

sends request to server
gets result back to client
What is ORB?
ORB = Object Request Broker

ORB is the core component of CORBA.

It:

locates remote objects
handles communication
transfers requests/responses
CORBA Architecture
Client
   ↓
ORB
   ↓
Remote Object
   ↓
Server

What is IDL?
IDL = Interface Definition Language

Used to define:

remote methods
parameters
return types

IDL is language independent.

Why IDL is used?

Because CORBA supports multiple languages.

IDL acts as common communication format.

Methods Defined

Meaning:
takes two float inputs
returns float result

C) Server.java
Purpose

Starts CORBA server and registers remote object.

Step-by-step Explanation
Step 1: Initialize ORB
ORB orb = ORB.init(args, null);

Starts CORBA runtime environment.

Step 2: Get RootPOA
POA rootpoa = POAHelper.narrow(...)

Gets reference to POA.

POA manages CORBA objects.

Step 3: Activate POA Manager
rootpoa.the_POAManager().activate();

Starts object request handling.

Step 4: Create Servant
CalculatorImpl calcImpl = new CalculatorImpl();

Creates actual calculator object.

Step 5: Convert to CORBA Object
servant_to_reference(calcImpl)

Converts Java object into CORBA object.

Step 6: Naming Service
resolve_initial_references("NameService")

Gets naming service reference.

Step 7: Bind Object
ncRef.rebind(path, href);

Registers object with name:

Calculator
Step 8: Wait for Requests
orb.run();

Server continuously waits for client requests.

D) Client.java
Purpose

Client accesses remote calculator methods.

Step-by-step
Step 1: Initialize ORB

CORBA initially returns generic object reference.

Viva Questions and Answers
Q1) What is CORBA?
Answer:

CORBA is a distributed object architecture that enables communication between objects running on different systems and languages.

Q2) What is ORB?
Answer:

ORB stands for Object Request Broker.
It manages communication between client and server objects.

Q3) Why is IDL used?
Answer:

IDL provides language-independent interface definitions so different programming languages can communicate.

Q4) What is Naming Service in CORBA?
Answer:

Naming Service stores object references and allows clients to locate remote objects using names.

Q5) What is POA?
Answer:

POA stands for Portable Object Adapter.
It connects servant objects with ORB and manages remote object requests.

Q6) What is narrowing in CORBA?
Answer:

Narrowing converts a generic CORBA object reference into a specific object type.

Q7) What is a servant in CORBA?
Answer:

Servant is the actual implementation of remote methods on server side.

Q8) What happens if server is not running?
Answer:

Client cannot locate remote object and communication fails.
