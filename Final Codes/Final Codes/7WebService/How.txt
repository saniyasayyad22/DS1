2) Server.java Explanation
Imports
import com.sun.net.httpserver.HttpServer;
Used to create HTTP server.

HttpHandler
Handles client requests.

HttpExchange
Contains request and response information.

InetSocketAddress
Defines IP address and port.

Main Method
public static void main(String[] args)
Program execution starts here.

Create HTTP Server
HttpServer server = HttpServer.create(new InetSocketAddress(8000), 0);
Creates web server on:
Port 8000

Create Context
server.createContext("/add", new MyHandler());
Creates endpoint:
/add
All requests to:
http://localhost:8000/add
handled by:
MyHandler

Executor
server.setExecutor(null);
Uses default thread executor.

Start Server
server.start();
Starts HTTP server.

MyHandler Class
static class MyHandler implements HttpHandler
Handles incoming requests.

handle() Method
public void handle(HttpExchange t)
Automatically called when client sends request.

Read Query Parameters
String query = t.getRequestURI().getQuery();
Reads:
a=10&b=5

Split Parameters
query.split("&");
Separates:
a=10b=5

Extract Numbers
Integer.parseInt(...)
Converts string into integer.

Perform Addition
int result = a + b;
Calculates sum.

Prepare Response
String response = "Sum = " + result;
Creates response string.

Send HTTP Status
t.sendResponseHeaders(200, response.length());
200 means:
Success

Send Response Data
OutputStream os = t.getResponseBody();
Gets response stream.

Write Response
os.write(response.getBytes());
Sends response to client.

Close Stream
os.close();
Closes connection.

Viva Questions and Answers
Q1) What is a web service?
Answer:
A web service is a software service that enables communication between applications over a network using protocols like HTTP.

Q2) What is HTTP?
Answer:
HTTP stands for HyperText Transfer Protocol and is used for communication between clients and web servers.

Q3) What is GET request?
Answer:
GET request is used to retrieve data from the server through URL parameters.

Q4) What does this program do?
Answer:
The client sends numbers to the web server, and the server performs addition and returns the result.

Q5) What is URL?
Answer:
URL is the address used to access a web service or resource.

Q6) What is the role of HttpHandler?
Answer:
HttpHandler processes incoming HTTP requests on the server.

Q7) What does status code 200 mean?
Answer:
HTTP status code 200 means request processed successfully.

Q8) Difference between client and server?
Answer:
Client requests services, while server provides services.

Q9) What is HttpURLConnection?
Answer:
It establishes HTTP communication between client and server.

Q10) Why port 8000 is used?
Answer:
Port 8000 is used as communication endpoint for the web server.

Q11) What are query parameters?
Answer:
Query parameters are values passed in URL like:
?a=10&b=5

Q12) Real-life examples of web services?
Answer:
Banking apps, weather apps, Google Maps, online shopping, food delivery systems, and social media platforms.

Short Viva Conclusion

This assignment demonstrates a distributed web service application where a client communicates with a server using HTTP requests. The server processes addition requests and returns responses through a simple REST-style web service.
