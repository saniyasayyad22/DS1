1) What this assignment is about?

This assignment implements the Token Ring Algorithm in a distributed system.

The goal is:

ensure mutual exclusion
allow only one process at a time to enter the critical section

Processes are connected in a logical ring, and a special message called TOKEN circulates among them.

Only the process holding the token can access the critical section.

The lab manual explains Token Ring Algorithm ensures mutual exclusion using a circulating token in a logical ring structure.

What problem does this solve?

In distributed systems:

many processes may try accessing shared resources simultaneously

Example:

printer
database
shared file
transaction system

Without control:

data corruption
race conditions
conflicts occur

Token Ring Algorithm solves this problem.

What is Mutual Exclusion?
Definition

Mutual exclusion means:

Only one process can enter critical section at a time.

What is Critical Section?

Critical section is:

Part of program where shared resource is accessed.

Example:

writing to file
updating database
using printer
What is Token Ring Algorithm?
Definition

Token Ring is a distributed mutual exclusion algorithm where:

processes form a logical ring
one unique token circulates continuously
only token holder can enter critical section
Main Idea
NO TOKEN → Cannot enter CS
HAS TOKEN → Can enter CS
Real-life Analogy

Suppose:
5 students share one microphone.

Rule:

only student holding microphone can speak

Microphone passes circularly.

Here:

Microphone = TOKEN

This avoids everyone speaking together.

Token Ring Working
Step 1

Processes arranged in ring.

Example:

P1 → P2 → P3 → P4 → P1
Step 2

One token circulates.

Step 3

Process receives token.

Step 4

Process enters critical section.

Step 5

After completion, token passed to next process.

Why Token Ring is used?

Used because:

guarantees mutual exclusion
avoids deadlock
avoids starvation
fair access to resources

Viva Questions and Answers
Q1) What is Token Ring Algorithm?
Answer:

Token Ring is a distributed mutual exclusion algorithm where processes form a logical ring and a unique token circulates among them. Only the token holder can enter the critical section.

Q2) What is mutual exclusion?
Answer:

Mutual exclusion ensures only one process accesses shared resources at a time.

Q3) What is a token?
Answer:

A token is a special permission message allowing a process to enter the critical section.

Q4) Why is token required?
Answer:

To prevent multiple processes from entering critical section simultaneously.

Q5) What is critical section?
Answer:

Critical section is the part of code where shared resources are accessed.

Q6) Why is ring structure used?
Answer:

Ring structure provides orderly token passing among processes.

Q7) What happens if token is lost?
Answer:

No process can enter critical section until token is regenerated.

Q8) Why only Process 1 starts token?
Answer:

To ensure only one token exists initially.

Q9) How does this code achieve mutual exclusion?
Answer:

Only process holding token enters critical section while others wait.

Q10) What is the role of sockets here?
Answer:

Sockets provide communication between distributed processes.

Q11) Why is while(true) used?
Answer:

Processes continuously wait and participate in token passing.

Q12) Why are delays added using Thread.sleep()?
Answer:

To simulate work and clearly show token movement.
